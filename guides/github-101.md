# GitHub 101: Your First Contribution

Never used GitHub before? This guide gets you from zero to your first pull request in under 15 minutes. No terminal. No coding. Just a browser.

---

## Step 1: Create a GitHub Account

1. Go to [github.com](https://github.com)
2. Click **Sign up**
3. Pick a username, enter your email, create a password
4. Verify your email
5. Done — you have a GitHub account

---

## Step 2: Find Something to Contribute

The easiest first contribution is fixing a typo or adding to a resource list.

1. Go to the [trillion-library repo](https://github.com/Trillion-Builders/trillion-library)
2. Browse through any file — README, guides, resource lists
3. Spot something you can improve? A typo, an unclear sentence, a missing resource?

---

## Step 3: Make Your Edit

1. Click on the file you want to edit
2. Click the **pencil icon** (✏️) in the top-right corner of the file
3. GitHub will tell you it's creating a "fork" — that's normal, click through
4. Make your edit in the text editor
5. Scroll down to **"Propose changes"**
6. Write a short title describing what you changed (e.g., "Fixed typo in README")
7. Click **Propose changes**

---

## Step 4: Create Your Pull Request

1. After proposing changes, you'll see a **"Create pull request"** page
2. The title and description should already be filled in
3. Click **Create pull request**
4. That's it — your contribution is submitted for review

---

## Step 5: Wait for Review

Someone from the team will review your pull request. They might:
- **Merge it** — your change is now part of the repo. Congratulations, you've contributed!
- **Request changes** — they might ask you to tweak something. No big deal, just make the edit.
- **Comment** — they might have a question or suggestion.

---

## What Just Happened?

You just:
1. **Forked** the repo (made your own copy)
2. **Edited** a file (made your change)
3. **Opened a pull request** (proposed your change to the main repo)
4. Someone **reviewed and merged** it (your change is now official)

That's the entire Git workflow. Everything else is just variations on this.
