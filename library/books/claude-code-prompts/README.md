# Claude Code Prompt Library

**Copy. Paste. Ship.**

This is a living collection of prompts that work — tested by builders in this community. You don't need to know how to code to use these. You need Claude Code, an idea, and the willingness to try.

Every prompt here has been used to build something real. When you find one that works for you, add it.

---

## How to Use This

1. Open Claude Code
2. Copy a prompt below
3. Replace anything in `[brackets]` with your specifics
4. Hit enter
5. Watch it build

That's it.

## Getting Started

**Set up a new project from scratch**
```
Build me a [personal link-in-bio page / countdown timer / simple dashboard].
Use Next.js and Tailwind CSS. Keep it simple and clean.
When it's done, tell me how to deploy it to Vercel.
```

**Understand a codebase you just inherited**
```
I just opened this project for the first time.
Walk me through what it does, how it's structured, and where the most important files are.
Explain it like I'm smart but new to coding.
```

**Start from an idea, not a spec**
```
I want to build [describe your idea in plain english].
I'm not a developer. Ask me 3 clarifying questions, then build a simple version I can run locally.
```

## Building Features

**Add something new to an existing project**
```
I want to add [feature description] to this project.
Look at the existing code first so it fits the style.
Build it step by step and tell me what you're doing.
```

**Build a page or screen**
```
Build a [settings page / dashboard / landing page / profile page] for this app.
Use the same design style as the rest of the project.
It should include: [list what you want on it].
```

**Connect to an API or external service**
```
I need to connect this app to [Stripe / a database / Twitter API / etc.].
Show me the simplest way to do it.
Use environment variables for any keys and tell me exactly what values I need to fill in.
```

**Add a form that does something**
```
Add a form that lets users [submit feedback / sign up for a waitlist / contact me].
When they submit it, [send me an email / save it to a database / show a success message].
```

## Debugging

**Something broke and I don't know why**
```
This stopped working. Here's the error:
[paste the exact error message]

Here's what I was doing when it broke:
[describe what you did]

Fix it.
```

**It builds but doesn't do what I want**
```
This code runs without errors but the result is wrong.
What I expected: [describe expected behavior]
What actually happens: [describe what happens]
Find the problem and fix it.
```

**I'm getting a weird error I don't understand**
```
I got this error and I don't know what it means:
[paste error]

Explain what's causing it in plain English, then fix it.
```

## Deploying

**Deploy to Vercel for the first time**
```
I've never deployed anything before.
Walk me through deploying this project to Vercel step by step.
Assume I have a Vercel account but haven't used the CLI.
```

**Set up environment variables**
```
This app needs environment variables to run.
Tell me exactly which ones I need, where to get them,
and how to add them to Vercel so the deployed version works.
```

**Something works locally but not in production**
```
This works on my computer but breaks when I deploy to Vercel.
Here's the error from the Vercel logs:
[paste logs]
Fix it.
```

## Learning / Understanding

**Explain what this code does**
```
Explain what this code does, line by line, like I'm smart but new to programming:
[paste code]
```

**Teach me how this works**
```
I want to understand how [authentication / databases / APIs / React / etc.] works.
Don't just tell me the theory — show me a simple working example
and explain what each part does.
```

**What should I learn next?**
```
I just built [describe what you built].
I understand [describe what you know].
What's the most useful thing I should learn next, and why?
```

**Translate developer jargon**
```
Someone used the phrase "[technical term]" and I don't know what it means.
Explain it in plain English and give me an example of it in action.
```

## Idea Generation

**Turn a rough idea into a buildable spec**
```
I have a rough idea: [describe it].
Turn this into a clear project brief — what it does, who it's for,
what the simplest version looks like, and what I'd need to build it.
```

**What could I build with what I know?**
```
I know how to [describe your current skill level].
Suggest 5 project ideas that would stretch me slightly
but are actually finishable in a weekend.
```

**Make something I built more useful**
```
I built [describe your project].
What are 3 ways I could make it significantly more useful or valuable
without rebuilding the whole thing?
```
